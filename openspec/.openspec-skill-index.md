## How to Use This Skill

### For OpenClaw AI Assistant

This skill is automatically loaded when you invoke OpenSpec commands:

```
/opsx:new [change-name]        # Start a new change
/opsx:ff                       # Create all planning artifacts
/opsx:apply                    # Implement tasks
/opsx:archive                  # Archive completed change
```

See [SKILL.md](./SKILL.md) for complete command reference.

### Manual Workflow

If you prefer not to use slash commands, you can manually create the structure:

1. Create `openspec/changes/<change-name>/` directory
2. Copy templates from `templates/` folder
3. Fill in the artifacts
4. Use the AI assistant to implement tasks

## Learning Path

### New to OpenSpec?
1. Start with [README.md](./README.md) for the big picture
2. Read the "Quick Start" section
3. Try the `/opsx:onboard` command for interactive tutorial

### Want to Understand the Methodology?
1. Read the "Core Concepts" section in [SKILL.md](./SKILL.md)
2. Study the "Artifact Dependencies" diagram
3. Review [EXAMPLES.md](./EXAMPLES.md) to see real workflows

### Ready to Build Something?
1. Initialize your project with `openspec init` (CLI) or create structure manually
2. Run `/opsx:new <your-feature>` to start
3. Use `/opsx:ff` or `/opsx:continue` to create artifacts
4. Run `/opsx:apply` to implement
5. Run `/opsx:archive` when done

## Philosophy Reminder

OpenSpec is built around four principles:

- **Fluid, not rigid** — No phase gates, work on what makes sense
- **Iterative, not waterfall** — Learn as you build, refine as you go
- **Easy, not complex** — Lightweight setup, minimal ceremony
- **Brownfield-first** — Works with existing codebases
