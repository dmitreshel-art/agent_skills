# Dense Векторы (Плотные)

## Что такое Dense Векторы?

Dense векторы — это фиксированной длины числовые векторы, где **каждое измерение несёт семантическую информацию**. В отличие от разрежных (sparse) векторов, где большинство измерений равны нулю, в плотных векторах почти все элементы активны.

## Ключевые характеристики

### Фиксированная длина
```python
# Пример 768-мерного dense вектора
dense_vector = [
    0.023, -0.041, 0.005, ..., 0.089  # 768 значений
]

# Длина: 768 измерений
```

### Почти все элементы активны
```python
# Sparse вектор (50,000 измерений)
sparse_vector = {
    "term1": 2.5,      # Активно
    "term2": 1.8,      # Активно
    "term3": 0.3,      # Неактивно (почти ноль)
    ...
    "term50000": 0.1  # Неактивно
}

# Dense вектор (768 измерений)
dense_vector = [
    0.012, -0.034, 0.001, ..., 0.099  # Почти все активны
]
```

### Высокая размерность
- Типично: 768, 1024, 1536, 2048, 3072 dimensions
- Определяет качество модели (большая размерность = лучше семантика)

### Семантическая близость
- Все измерения участвуют в вычислении сходства
- Косинус similarity: `cos(angle) = dot(A,B) / (|A|*|B|)` — угол между векторами
- Чем больше активных элементов с похожими значениями, тем более близкие векторы

## Преимущества

### ✅ Богатая семантика
- Все измерения несут информацию
- Хорошо captures концепции и связи
- Идеально для сложного контента

### 🎯 Подходит для многии задач
- Семантический поиск
- RAG (Retrieval-Augmented Generation)
- Вопросно-ответные системы

### 📊 Простота в использовании
- Единая размерность для всех векторов
- Простая индексация (HNSW)
- Эффективное вычисление сходства

## Недостатки

### ❌ Высокая вычислительная цена
- **Память:** Большая размерность = много памяти для хранения
- **Поиск:** Дольше обрабатывать большие векторы
- **Вычисления:** Труднее вычислять сходство в высокомерном пространстве
- **Storage:** Больше места для хранения векторов

### ⚠️ Универсальная, но неоптимизированная
- Хорошая для general-purpose задач
- Не специализируется на конкретные типы данных
- Может быть избыточной для узких задач

### 🔧 Можно оптимизировать

```python
# Уменьшите размерность через PCA или автоэнкодер
# Используйте скалирование (quantization) для снижения памяти
# Усекайте неактивные измерения
# Используйте product quantization (8-bit floats)
```

## Sparse vs Dense

| Характеристика | Sparse Векторы | Dense Векторы |
|---------------|---------------|-------------------|
| Размерность | Высокая (50,000+) | Средняя (768-1024) |
| Активность | Разрежная (~5%) | Плотная (~90%) |
| Память | Эффективнее | Большая |
| Семантика | Ключевые слова | Полная информация |
| Использование | Поиск по ключам | Семантический поиск |
| Вычисления | Быстрее | Медленнее |
| Лучше для | Текст/ключевые слова | Изображения/аудио |

## Когда использовать Dense Векторы

### Подходит для:
- **Общего назначения** — когда нужны богатые семантические представления
- **Семантический поиск** — по смыслу, а не по точному совпадению
- **Рекомендательные системы** — когда важны все детали и связи
- **Вопросно-ответные системы** — QA, чатботы
- **Изображения** — когда визуальная семантика важна

### Не рекомендуется для:
- **Текстовые задачи** — sparse векторы эффективнее
- **Ключевой поиск** — избыточны, sparse векторы лучше
- **Ограниченные ресурсы** — когда важна эффективность

## Dense Векторы в ZVEC

ZVEC поддерживает dense векторы с типом `zvec.DataType.VECTOR_FP32`:

```python
import zvec

# Определите схему коллекции
schema = zvec.CollectionSchema(
    name="my_collection",
    vectors=zvec.VectorSchema("embedding", zvec.DataType.VECTOR_FP32, 768)  # 768 dimensions
)

# Создайте коллекцию
collection = zvec.create_and_open(path="./dense_collection", schema=schema)

# Вставьте документ
collection.insert(
    zvec.Doc(
        id="doc_1",
        vectors=zvec.VectorValue.dense(
            vector=[0.012, -0.034, 0.001, ..., 0.089]  # 768-мерный вектор
        )
    )
)
```

## Особенности Dense Векторов

### 1. Единая размерность
```python
# Все векторы в коллекции имеют одинаковый размер
# Это упрощает вычисления сходства и индексацию
```

### 2. Полная активность
```python
# Почти все измерения вносят вклад в семантику
# Результаты поиска более точны и релевантны
```

### 3. Эффективная индексация
```python
# ZVEC использует HNSW для быстрых приближённых запросов
# Отлично работает с большими объёмами данных
```

## Примеры использования

### Семантический поиск

```python
# Поиск похожих документов
query_vector = zvec.encode("векторный поиск для AI приложений")

results = collection.query(
    zvec.VectorQuery("embedding", vector=query_vector, topk=10)
)

for result in results:
    print(f"Document: {result.id}, Score: {result.score}")
```

### Рекомендательные системы

```python
# Поиск документов для рекомендации
user_profile_vector = zvec.encode("профиль пользователя")

results = collection.query(
    zvec.VectorQuery("embedding", vector=user_profile_vector, topk=5)
)

# Показать наиболее релевантные документы
for result in results:
    print(f"Recommended: {result.id}")
```

## Метрики сходства

### Cosine Similarity (Косинусное сходство)

```python
import numpy as np

def cosine_similarity(vec1, vec2):
    """
    Косинусное сходство (от -1 до 1)
    Чем ближе к 1, тем больше сходство
    """
    dot_product = np.dot(vec1, vec2)
    norm1 = np.linalg.norm(vec1)
    norm2 = np.linalg.norm(vec2)
    return dot_product / (norm1 * norm2)

# Пример
similarity = cosine_similarity(vector_a, vector_b)
# 0.95 = очень похожие
# 0.0 = ортогональные (полностью разные)
# 0.7 = умеренно похожие
```

### Dot Product (Скалярное произведение)

```python
def dot_product(vec1, vec2):
    """
    Скалярное произведение (направленность не важна)
    Проекция одного вектора на другой
    """
    return sum(a * b for a, b in zip(vec1, vec2))

# Пример
score = dot_product(query_vector, candidate_vector)
# Чем выше, тем больше похожие (в определённом направлении)
```

### Euclidean Distance (Евклидово расстояние)

```python
def euclidean_distance(vec1, vec2):
    """
    Прямое расстояние между двумя точками
    Чем ближе, тем более похожие
    """
    return np.sqrt(sum((a - b) ** 2 for a, b in zip(vec1, vec2)))

# Пример
distance = euclidean_distance(vector_a, vector_b)
# 0.05 = очень похожие
# 2.5 = совершенно разные
# 1.0 = умеренно похожие
```

## Производительность Dense Векторов

### Вычислительные затраты
- **Память:** ~3 KB на 768-мерный вектор (FP32)
- **Сходство:** ~1000 FLOPS для cosine similarity
- **Индексация:** O(log N) для HNSW

### Масштабируемость
- **Маленькие данные (<1M):** Отлично работает
- **Средние данные (1-10M):** Хорошо масштабируется
- **Большие данные (10-100M):** Требует оптимизации

## Best Practices

### 1. Выбор размерности

```python
# Подход: начните с меньшей размерности и увеличивайте по необходимости
# Руководство: 768-1024 для общего назначения
# Для изображений: 1536-2048 dimensions (CLIP, DINO)
# Для аудио: 512-1024 dimensions (wav2vec, VGGish)
```

### 2. Оптимизация производительности

```python
# Используйте скалирование для снижения памяти
# Product quantization: 8-bit вместо 32-bit
# Векторные базы: используйте ZVEC для больших данных

### 3. Индексация

```python
# HNSW — Hierarchical Navigable Small World
# Параметры M (количество соседей) = 16-32
# EF (Exploratory Factorization) для очень больших данных
```

## Заключение

Dense векторы — это мощный инструмент для задач, требующих богатых семантических представлений. Они идеально подходят для семантического поиска, рекомендательных систем и приложений, где важна точность и связность данных.

ZVEC с его efficient индексацией делает dense векторы практичными даже на миллиарды векторов.

Для сложных задач рекомендую использовать sparse векторы + фильтры для лучшей производительности.
