# ZVEC Основы

## Что такое ZVEC?

ZVEC — это open-source, in-process векторная база данных от Alibaba для быстрого семантического поиска. Это лёгкая библиотека, которая работает непосредственно в памяти вашего процесса — без серверов, демонов или внешней инфраструктуры.

## Ключевые особенности

### ⚡ Сверхбыстрая скорость
- **Миллионы векторов за миллисекунды** — экстремально быстрые поиски
- **Низкая задержка** — sub-millisecond latency для ответов
- **Производительность** — оптимизировано для production workloads

### 🧩 Прост в использовании
```bash
pip install zvec
```
- Без конфигурации
- Без настройки серверов
- Начните искать за секунды
- Работает из коробки

### ✨ Dense + Sparse векторы
- **Dense векторы** — фиксированной длины (обычно 768 dimensions), почти все элементы активны
- **Sparse векторы** — высокой размерности (обычно 50,000+ словарь), только ключевые слова активны
- **Native поддержка** — multi-vector queries в одном вызове
- **Гибкость** — работайте с любым типом векторов

### 🎯 Гибридный поиск
- **Семантическое сходство** — находите по смыслу, а не по точному совпадению
- **Структурированные фильтры** — комбинируйте семантику с условиями (SQL-подобный синтаксис)
- **Точные результаты** — условия для более релевантных ответов

### 📱 Работает везде
- **In-process** — работает где угодно: Jupyter notebooks, серверы, CLI инструменты, даже edge устройства
- **Python 3.10+** — современная поддержка
- **Node.js** — поддержка через `@zvec/zvec`
- **Кросс-платформенный** — Linux, macOS, ARM64

## Как работает ZVEC?

### Архитектура

```
┌───────────────────────┐
│  Входные данные              │
│  (текст, изображения, аудио)   │
│         ↓                     │
│  Embedding модель             │
│  (OpenAI, CLIP, etc.)       │
│         ↓                     │
│  Векторные представления   │
│  (Dense/Sparse embeddings) │
│         ↓                     │
│  ZVEC коллекция             │
│  (in-process база данных)  │
│         ↓                     │
│  Эффективные индексы         │
│  (HNSW, IVF)              │
│         ↓                     │
│  Поиск и выдача           │
└───────────────────────┘
```

### Пошаговый процесс

1. **Создание коллекции** — определите схему
2. **Вставка документов** — добавьте данные с векторами
3. **Поиск** — создайте векторный запрос, получите результаты

- Статьи, документация, знания

**Изображения**
- Фото продуктов, лица, сцены

**Аудио**

## Ключевые концепции

### Векторные embeddings

**Что это:** Векторные embeddings — это списки чисел, которые генерируются AI моделями для захвата семантической сущности данных (текст, изображения, аудио).

**Как работают:**
1. **Обучение** — Модель анализирует огромные объёмы данных и учится находить паттерны
2. **Векторизация** — Каждый вход (текст, изображение) превращается в точку в высокомерном пространстве
3. **Сравнение** — Похожие данные становятся близкими точками в этом пространстве
4. **Поиск** — Находят ближайшие векторы к запросу = семантически похожие результаты

### Dense vs Sparse векторы

| Характеристика | Dense Векторы | Sparse Векторы |
|---------------|----------------|-----------------|
| Длина | Фиксированная (384, 768, 1536) | Высокая (50,000+) |
| Размерность | Почти все активны | Большинство нулевые |
| Семантика | Понимает контекст | Фокус на ключевые слова |
| Использование | RAG, семантический поиск | Поиск по ключевым словам |
| Память | Больше для хранения | Меньше для хранения |
| Вычисления | Быстрее | Медленнее, но эффективнее |

**Примеры:**

**Dense вектор:**
```python
# 768-мерный вектор из текста "король живёт во дворце"
dense_vector = [0.012, -0.023, 0.045, ..., 0.009]  # 768 значений

# Этот вектор понимает, что текст про монарха, дворец и жизнь
# Похожие тексты ("царь живёт", "монарх в замке") создадут похожие векторы
```

**Sparse вектор:**
```python
# Sparse вектор над словарём из 50,000 слов
sparse_vector = {
    "король": 2.31,       # "king" имеет высокий вес
    "живёт": 1.85,      # "lives" имеет вес
    "во": 0.92,          # "in" имеет низкий вес
    "дворце": 1.45,      # "palace" имеет высокий вес
    # ... остальные 49,996 размерностей неявно равны 0
}

# Размер вектора: 50,000 (по словарю)
# Активно: только ~5-7 слов ("король", "живёт", "во", "дворце")
# Оставшиеся 49,993 размерности неявно равны 0

# Использование: поиск по ключевым словам
# Результат: найдите документы, где встречаются эти слова
```

### Эффективные индексы

ZVEC использует оптимизированные алгоритмы для быстрого поиска:

- **HNSW (Hierarchical Navigable Small World)** — для быстрого поиска на больших данных
- **IVF (Inverted File Index)** — для точного поиска
- **EF Construction** — для компактного хранения

## Применения

### Семантический поиск (RAG)

ZVEC идеально подходит для RAG:

1. **Храните документы** — статьи, документация, знания
2. **Создайте embeddings** — превратите документы в векторы
3. **Находите похожие** — при запросе найдите релевантные документы
4. **Питайте LLM** — дайте контекст сгенерируйте ответ

### Поиск по изображениям

```python
# Изображение → вектор
image_vector = model.encode_image("photo_of_product.jpg")

# Поиск по сходству
results = zvec_collection.query(image_vector, topk=10)

# Результат: визуально похожие продукты
```

## Установка

### Требования

- **Python:** 3.10 или новее
- **Node.js:** Современный браузер
- **Память:** Достаточно для вашей коллекции

### Quickstart

```bash
# 1. Установите ZVEC
pip install zvec

# 2. Проверьте установку
python -c "import zvec; print('ZVEC installed successfully!')"

# 3. Запустите быстрый тест
python -c "
import zvec

# Создайте схему
schema = zvec.CollectionSchema(
    name='example',
    vectors=zvec.VectorSchema('embedding', zvec.DataType.VECTOR_FP32, 4)
)

# Создайте коллекцию
collection = zvec.create_and_open(path='./zvec_data', schema=schema)

# Вставьте документы
collection.insert([
    zvec.Doc(id='doc_1', vectors={'embedding': vector1}),
    zvec.Doc(id='doc_2', vectors={'embedding': vector2}),
    zvec.Doc(id='doc_3', vectors={'embedding': vector3})
])

# Поиск по сходству
results = collection.query(
    zvec.VectorQuery('embedding', vector=query_vector, topk=10)
)

# Результаты
for result in results:
    print(f'ID: {result.id}, Score: {result.score}')
"
```

## Производительность

### Масштабируемость

- **Миллиарды векторов** — ZVEC обрабатывает миллиарды векторов

### Эффективность

- **In-process** — Нет сетевого overhead
- **Эффективные индексы** — HNSW, IVF для быстрого поиска
- **Оптимизация памяти** — Умное управление памятью

## Настройки

### Оптимизация

```python
# Оптимизируйте коллекцию для производительности
collection.optimize()

# Проверьте статистику
print(collection.stats())

# Настройте параметры индекса

    }

```

## Концепции для понимания

### Косинусное сходство

```python
# Измерение угла между векторами
import numpy as np

def cosine_similarity(vec1, vec2):
    """Косинусное сходство (от -1 до 1)"""
    dot_product = np.dot(vec1, vec2)
    norm1 = np.linalg.norm(vec1)
    norm2 = np.linalg.norm(vec2)
    return dot_product / (norm1 * norm2)
```

**Интерпретация:**
- 1.0 — идентичные
- 0.9 — очень похожие
- 0.7 — похожие
- 0.0 — полностью разные

### Dot Product (Скалярное произведение)

```python
def dot_product(vec1, vec2):
    """Скалярное произведение"""
    return sum(a * b for a, b in zip(vec1, vec2))
```

**Интерпретация:**
- Положительный — векторы направлены в одну сторону
- Отрицательный — векторы направлены в противоположные стороны
- Ноль — ортогональны

### Euclidean Distance (Евклидово расстояние)

```python
def euclidean_distance(vec1, vec2):
    """Евклидово расстояние"""
    return np.sqrt(sum((a - b) ** 2 for a, b in zip(vec1, vec2)))
```

**Интерпретация:**
- 0 — идентичные
- Маленькое расстояние — очень похожие
- Большое расстояние — мало похожие

## Поиск

### Основные операции

#### 1. Создание коллекции

```python
import zvec

# Создайте схему
schema = zvec.CollectionSchema(
    name='my_collection',
    vectors=zvec.VectorSchema('embedding', zvec.DataType.VECTOR_FP32, 768)
)

collection = zvec.create_and_open(path='./my_collection', schema=schema)
```

#### 2. Вставка документов

```python
# Простая вставка
collection.insert(zvec.Doc(
    id='doc_1',
    vectors={'embedding': vector1}
))

# Массовая вставка
documents = [
    zvec.Doc(id=f'doc_{i}', vectors={'embedding': vec})
    for i, vec in enumerate(vectors)
]
collection.insert(documents)
```

#### 3. Поиск

```python
# Базовый поиск
results = collection.query(
    zvec.VectorQuery('embedding', vector=query_vector, topk=10)
)

# Фильтрованный поиск
results = collection.query(
    zvec.VectorQuery('embedding', vector=query_vector, topk=10),
    filter="publish_year >= 2020"  # Только новые документы
)

# Multi-vector поиск
results = collection.query(
    zvec.VectorQuery(
        'embedding',
        vector=[vec1, vec2, vec3]  # 3 разных вектора
    ),
    topk=5
)
```

## Резервное копирование

```python
# Экспорт коллекции
collection.export_backup(path='./backup')

# Восстановление из резервной копии
collection = zvec.open('./backup')
```
