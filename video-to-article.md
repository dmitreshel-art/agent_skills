# video-to-article

Генерация подробной статьи на основе видео.

## Принцип

**Максимально подробный пересказ** — статья должна давать эффект "я сам посмотрел видео". Минимальная суммаризация, сохраняем детали, примеры, рассуждения.

## Входные данные

- `url` — URL видео (YouTube)
- `language` — язык статьи (по умолчанию русский)
- `detail_level` — уровень детализации: `full` (по умолчанию) / `summary` (кратко)

## Что вырезаем

- Организационные моменты (приветствия, анонсы, реклама, благодарности)
- Технические проблемы (звук, связь, перебои)
- Паузы, filler words ("э-э", "ну", "как бы")
- Off-topic дискуссии (не относящиеся к теме видео)

## Что сохраняем

- Все тезисы и аргументы
- Примеры и кейсы из практики
- Цитаты спикера (в блоке цитат)
- Логику рассуждений (потому что → следовательно)
- Детали реализации (технические, процессные, методологические)
- Ответы на вопросы из зала (Q&A)
- Нюансы, оговорки, рекомендации

## Структура статьи

### 1. Заголовок и мета
```
# Название видео (или статьи)

**Автор:** Имя Фамилия
**Источник:** Название канала/конференции
**Дата:** DD.MM.YYYY
**Длительность:** XX мин
```

### 2. Контекст
1-2 предложения: где, кто, о чём видео.

### 3. Основная часть
Хронологический пересказ с подзаголовками по темам.

Формат:
- Таймкоды в тексте: `[MM:SS]`
- Цитаты в блоках:
  ```
  > «Прямая цитата спикера»
  ```
- Таблицы по ходу текста при наличии структурированных данных
- Изображения встроены в соответствующие места

Объём: 70-90% от транскрипции.

### 4. Ключевые кадры
Встроены в текст статьи с подписями:
```
[Изображение: MM:SS — Описание что на кадре]
```

### 5. Q&A (если были вопросы)
Вопросы и ответы в формате:
```
**Вопрос:** Текст вопроса

**Ответ:** Текст ответа
```

### 6. Итоги (опционально)
Краткое резюме ключевых выводов (только если requested).

## Процесс выполнения

1. **Получить метаданные видео**
   - Название, автор, канал, дата, длительность
   - Проверить наличие презентации в описании

2. **Получить транскрипцию**
   - Извлечь полный transcript видео
   - Сохранить тайминги для кадрирования

3. **Извлечь ключевые кадры**
   - Определить таймкоды смены тем
   - Извлечь кадры в эти моменты
   - Опционально: подписи к кадрам

4. **Написать статью**
   - Хронологический подробный пересказ
   - Встроить таймкоды `[MM:SS]`
   - Встроить кадры
   - Оформить цитаты блоками
   - Построить таблицы

5. **Создать DOCX**
   - Использовать python-docx
   - Встроить изображения
   - Форматирование: заголовки, цитаты, таблицы

6. **Отправить результат**
   - DOCX-файл в чат

## Пример вызова

```
Сделай статью из видео https://youtu.be/XXXXXXXX
```

или с параметрами:

```
Сделай статью из видео https://youtu.be/XXXXXXXX с подробным пересказом
```

## Формат вывода

DOCX-файл с названием: `{название_видео}_article.docx`

Размер статьи: ~70-90% от объёма транскрипции.
