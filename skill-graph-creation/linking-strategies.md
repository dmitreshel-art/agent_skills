# Связи между навыками

Связи — это то, что делает граф графом. Без связей навыки — это просто папка файлов. Хорошые связи делают знания проходимыми и контекстуальными.

## Типы связей

### 1. Иерархические (родитель → потомок)

Связи от базовых концепций к продвинутым.

**Правило:** От базового к продвинутому.

### 2. Ассоциативные (смежные концепции)

Связи между смежными концепциями, которые часто используются вместе.

**Правило:** Связывайте смежные концепции.

### 3. Причинно-следственные (X вызывает Y)

Связи показывающие причинно-следственные отношения.

**Правило:** Показывайте причинно-следственные связи.

## Wiki-links синтаксис

### Базовый формат

```markdown

```

**Пример:**
```markdown
См. [[content-writing]] для деталей.
```

### Ссылки в тексте

**Хорошо (контекстуально):**
```markdown
Перед написанием кода, [[graph-structure]] поможет организовать файлы.
```

**Плохо (только ссылка):**
```markdown
См. также: content-writing.md
```

**Правило:** Ссылки должны нести смысл.

### Ссылки в списках

```markdown
## Related Skills

- [[graph-structure]] — Организация файлов
- [[naming-conventions]] — Правила именования
- [[validation]] — Проверка качества
```

**Правило:** Списки для навигации, текст для контекста.

## Стратегии связывания

### 1. Принципы → Примеры

В описании принципов приводите примеры реализации.

**Где ссылка:** В описании принципа.

### 2. Ошибки → Связанные концепции

В описании решений ссылайтесь на связанные концепции.

**Где ссылка:** В объяснении решения.

### 3. Интеграция → Компоненты

В Integration секции связывайте навыки.

**Где ссылка:** В Integration секции.

## Плотность связей

### Сколько связей на навык?

| Размер навыка | Минимум | Оптимум | Максимум |
|-------------|---------|----------|----------|
| Маленький (300-500 слов) | 2-3 | 4-6 | 8 |
| Средний (500-1000 слов) | 4-6 | 6-10 | 15 |
| Большой (1000-1500 слов) | 6-8 | 10-15 | 20 |

**Правило:** 6-10 связей — золотая середина.

### Бидирекционные связи

**Хорошо (обе стороны):**
```markdown
# content-writing.md

Связан с [[graph-structure]] — Структура влияет на написание
```

```markdown
# graph-structure.md

Связан с [[content-writing]] — Написание требует структуры
```

**Плохо (одностороннее):**
```markdown
# content-writing.md

Связан с [[graph-structure]]
```

**Правило:** Где возможно, связывайте обе стороны.

## Создание кластеров

### Когда использовать MOC (Maps of Content)?

**Симптомы для MOC:**
- 15+ навыков в графе
- INDEX.md становится перегруженным
- Естественные кластеры навыков

### Структура MOC

```markdown
# psychology-moc.md

Map of Content для психологических навыков

## Core

- [[content-writing]] — Написание навыков
- [[naming-conventions]] — Именование файлов
```

**Правило:** MOC для организации, INDEX для входа.

### Связи в MOC

```markdown
# INDEX.md

## Topic MOCs

-  — Все про написание
-  — Все про структуру
```

**Правило:** MOC не заменяет связи между навыками.

## Частые ошибки

### 1. Слишком мало связей

**Симптом:** Навык имеет 0-2 связи
**Проблема:** Агент не найдёт связанную информацию
**Решение:** Добавьте связи в Related секцию

### 2. Слишком много связей

**Симптом:** Навык имеет 20+ связей
**Проблема:** Перегруженность, сложно навигировать
**Решение:** Создайте MOC или разбейте навык

### 3. Мёртвые связи

**Симптом:** Ссылка которая никогда не используетса
**Проблема:** Ухудшение навигации
**Решение:** Удалите или обновите связь

### 4. Циклические связи

**Симптом:** A → B → C → A
**Проблема:** Бесконечный цикл
**Решение:** Это нормально для кластеров, но избегайте глубоких циклов

## Лучшие практики

### Правило 3-х кликов

**Цель:** Агент может дойти до любой информации за ≤3 клика

**Проверка:**
1. INDEX.md → целевой навык (1 клик)
2. Навык → Related секция (1 клик)
3. Related → целевая информация (1 клик)

**Если >3 клика:** Пересмотрите навигацию.

### Связи в контексте

**Плохо (безлично):**
```markdown
См. [[validation]] для деталей.
```

**Хорошо (в контексте):**
```markdown
Перед завершением, [[validation]] поможет гарантировать качество.
```

**Правило:** Объясняйте когда использовать связанный навык.

### Правило 80/20 для связей

- **80% связей:** Внутри вашего графа
- **20% связей:** На другие графы или внешние ресурсы

**Почему:** Фокус на внутренней связности.

## Wisdom

> "A link is a promise of relevance. Every wiki-link is a promise: 'If you need this, here's more."

Делайте честные обещания.
Связывайте только когда связь реальна.
Связывайте когда связь полезна.

Лучшая ссылка = контекстуально осмысленая.
Плохая ссылка = формальный список без смысла.
