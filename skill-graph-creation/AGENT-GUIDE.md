# Руководство для агента по использованию графов навыков

## Когда использовать графы навыков

Когда пользователь спрашивает о знаниях в конкретной области:

1. Проверьте, существует ли граф навыков для этой темы
2. Если да → используйте граф
3. Если нет → предложите создать новый граф

## Как работать с графом

### 1. Начните с INDEX.md

```python
# Псевдокод для агента
if user_asks_about_topic(topic):
    graph_path = f"skill-graphs/{topic}/INDEX.md"
    if exists(graph_path):
        index_content = read_file(graph_path)
        # Сканируйте YAML frontmatter
        # Покажите пользователю доступные навыки
    else:
        suggest_creating_graph(topic)
```

### 2. Сканируйте YAML frontmatter

Для каждого связанного навыка:
```yaml
---
description: "Краткое описание"
version: "1.0.0"
tags: ["tag1", "tag2"]
related: ["skill-1", "skill-2"]
---
```

Агент читает только frontmatter, чтобы понять:
- Что этот навык о чём
- Какие навыки связаны с ним
- Какие теги применимы

### 3. Следуйте по релевантным вики-ссылкам

В тексте используются вики-ссылки:
```markdown
См. [[risk-management]] для деталей.
```

Агент:
- Сканирует текст для поиска [[wiki-links]]
- Следует только по релевантным ссылкам
- Пропускает неважные разделы

### 4. Прогрессивное раскрытие

```
INDEX (индекс) → Описания (frontmatter) → Ссылки → Разделы → Полный контент
```

Большинство решений принимается до чтения полного файла.

## Пример работы

### Пользовательский запрос

"Расскажи про управление рисками в трейдинге"

### Процесс агента

1. **Находит граф:**
   ```
   skill-graphs/trading-graph/INDEX.md
   ```

2. **Читает INDEX.md:**
   - Находит `[[risk-management]]` в списке основных концепций
   - Видит, что это ядро системы

3. **Сканирует frontmatter risk-management.md:**
   ```yaml
   description: "Управление рисками и позициями в трейдинге"
   related: ["position-sizing", "risk-reward-ratio", "stop-loss-strategies"]
   ```

4. **Решает, что нужно:**
   - Основная концепция из risk-management.md
   - Связанные концепции для полноты

5. **Читает только нужные разделы:**
   - Core Principles из risk-management.md
   - Основные формулы из position-sizing.md
   - Обзор R:R из risk-reward-ratio.md

6. **Предоставляет целевые знания:**
   - Не весь файл
   - Только релевантные части
   - Связанные концепции

## Паттерн обнаружения

Графы навыков применяют тот же паттерн, что и обычные навыки:

1. **Сканируйте YAML frontmatter** для быстрого понимания
2. **Следуйте по вики-ссылкам** для связей
3. **Загружайте только нужные разделы** для эффективности

## Создание нового графа

### Шаг 1: Индекс

```markdown
---
description: "Описание графа"
---

# Название графа

## Основные концепции
- [[skill-1]] — Описание
- [[skill-2]] — Описание
```

### Шаг 2: Навыки

```markdown
---
description: "Описание навыка"
related: ["skill-1", "skill-2"]
---

# Название навыка

См. [[skill-1]] для деталей.
```

### Шаг 3: Связи

Используйте вики-ссылки в тексте:
- Для связей: `См. [[other-skill]]`
- В списках: `- [[skill-name]]`

## Лучшие практики

### 1. Один навык = одна идея

Хорошо:
- `risk-management.md` — всё про управление рисками

Плохо:
- `trading-everything.md` — всё сразу

### 2. Вики-ссылки несут смысл

Хорошо:
```markdown
Before entering a trade, check [[risk-management]] rules.
```

Плохо:
```markdown
See also: [[risk-management]]
```

### 3. Используйте YAML frontmatter

Всегда включайте:
- `description` — краткое описание
- `tags` — теги для поиска
- `related` — ссылки на связанные навыки

Добавьте в `AGENTS.md`:

```markdown
## Skill Graphs

When user asks about domain knowledge:
1. Check skill-graphs/ for relevant graphs
2. Read INDEX.md of relevant graph
3. Scan YAML frontmatter of related skills
4. Follow relevant wiki-links
5. Provide targeted knowledge, not entire files
6. Maintain progressive disclosure (index → descriptions → content)
```

## Troubleshooting

### Проблема: Граф не найден

**Решение:** Предложите создать новый граф
- Спросите пользователя, какие навыки нужны
- Создайте INDEX.md
- Добавьте основные навыки

### Проблема: Слишком много информации

**Решение:** Используйте прогрессивное раскрытие
- Начните с INDEX.md
- Сканируйте только frontmatter
- Читайте только нужные разделы

### Проблема: Слишком мало информации

**Решение:** Следуйте по вики-ссылкам
- Найдите связанные навыки
- Читайте только релевантные части
- Не читайте весь файл целиком

## Резюме

Графы навыков — это мощная система для структурированных знаний. Агент должен:

1. Начинать с INDEX.md
2. Сканировать YAML frontmatter
3. Следовать по релевантным вики-ссылкам
4. Предоставлять целевые знания
5. Не перегружать пользователя информацией

Это отличается от обычных навыков тем, что:
- Знания организованы в граф, а не список
- Агент перемещается по структуре, а не читает файлы целиком
- Прогрессивное раскрытие вместо полного чтения
- Релевантность через вики-ссылки, а не теги
