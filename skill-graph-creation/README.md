# Графы навыков для OpenClaw

## Что это такое?

**Графы навыков (Skill Graphs)** — это система взаимосвязанных знаний, где каждый файл — один навык, а вики-ссылки между ними создают проходимый граф знаний.

Вместо одного большого файла с инструкциями, агент получает структурированную сеть знаний, по которой можно перемещаться, находя именно то, что нужно для текущей задачи.

---

## Структура

```
skill-graphs/
├── README.md                 (этот файл)
└── trading-graph/           (пример: граф трейдинга)
    ├── INDEX.md             (точка входа)
    ├── risk-management.md
    ├── market-psychology.md
    ├── position-sizing.md
    └── emotional-control.md
```

---

## Как это работает

### 1. YAML Frontmatter

Каждый файл содержит метаданные в начале:

```yaml
---
description: "Краткое описание"
version: "1.0.0"
tags: ["tag1", "tag2"]
related: ["other-skill", "another-skill"]
---
```

Это позволяет агенту быстро понять, о чём файл, не читая его целиком.

### 2. Вики-ссылки

Ссылки на другие навыки встроены в текст:

```markdown
См. [[risk-management]] для деталей.
```

Агент следует по релевантным ссылкам, пропуская неважное.

### 3. Прогрессивное раскрытие

```
INDEX (индекс) → Описания (frontmatter) → Ссылки → Разделы → Полный контент
```

Большинство решений принимается до чтения полного файла.

## Как использовать

### Для агента

Когда вам нужны знания о конкретной теме:

```
1. Агент читает INDEX.md графа
2. Сканирует YAML frontmatter связанных файлов
3. Следует по релевантным вики-ссылкам
4. Загружает только нужные разделы
```

### Для пользователя

1. **Создайте новый граф:**
   ```bash
   mkdir -p skill-graphs/your-graph-name
   ```

2. **Создайте INDEX.md** — точку входа с навигацией

3. **Добавьте навыки** с YAML frontmatter и вики-ссылками

4. **Свяжите файлы** через [[wiki-links]]

## Создание нового графа навыков

### Шаг 1: Индексный файл

Создайте `INDEX.md`:

```markdown
---
description: "Точка входа в граф"
tags: ["main-topic"]
related: []
---

# Название графа

## Основные концепции
- [[skill-1]] — Описание
- [[skill-2]] — Описание

## Продвинутые темы
- [[skill-3]] — Описание
```

### Шаг 2: Файлы навыков

Каждый навык — отдельный файл:

```markdown
---
description: "Описание навыка"
tags: ["tag1", "tag2"]
related: ["skill-1", "skill-2"]
---

# Название навыка

См. [[skill-1]] для связанных концепций.
```

### Шаг 3: Связи

Используйте вики-ссылки для создания связей:
- В тексте: `См. [[other-skill]]`
- В списках: `- [[skill-name]]`
- В таблицах: `[[skill-name]]`

## Преимущества перед обычными навыками

| Обычные навыки | Графы навыков |
|---------------|---------------|
| Один файл | Сеть файлов |
| Ограниченная глубина | Неограниченная через ссылки |
| Инжиниринг контекста | Агент перемещается по структуре |
| Следование инструкциям | Понимание предметной области |

---

## Лучшие практики

### 1. Один навык = одна идея

Хорошо:
- `risk-management.md` — всё про управление рисками
- `emotional-control.md` — техники контроля эмоций

Плохо:
- `trading-everything.md` — всё сразу

### 2. Используйте YAML frontmatter

Всегда включайте метаданные:
- `description` — краткое описание
- `tags` — теги для поиска
- `related` — ссылки на связанные навыки

### 3. Ссылки должны нести смысл

Хорошо:
```markdown
Before entering a trade, check [[risk-management]] rules.
```

Плохо:
```markdown
See also: [[risk-management]]
```

### 4. MOC (Maps of Content)

Когда граф становится большим, используйте MOC для организации подтем:

```markdown
## Core MOCs

- [[psychology]] — все про психологию
- [[analysis]] — все про анализ
```

## Интеграция с AGENTS.md

Добавьте в `AGENTS.md`:

```markdown
## Skill Graphs

When user asks about knowledge in a domain:
1. Check if there's a skill-graph for this topic
2. Read the INDEX.md file
3. Scan YAML frontmatter of related files
4. Follow relevant wiki-links
5. Provide targeted knowledge, not entire files
```

## Идеи для новых графов

### 1. Граф разработки
- `clean-code.md` — принципы чистого кода
- `testing.md` — тестирование
- `debugging.md` — отладка

### 2. Граф медицины
- `diagnosis.md` — диагностика
- `treatment.md` — лечение
- `symptoms.md` — симптомы

### 3. Граф маркетинга
- `seo.md` — SEO
- `content-marketing.md` — контент-маркетинг
- `social-media.md` — соцсети

## Сравнение с arscontexta

| Характеристика | arscontexta | OpenClaw Skill Graphs |
|---------------|-------------|------------------------|
| Платформа | Claude Code | OpenClaw |
| Команды | `/learn`, `/reduce` | Чтение файлов |
| Организация | Автоматическая | Ручная |
| Структура | Рекурсивный граф | Индекс + файлы |

OpenClaw версия более простая и ручная, но даёт полный контроль.
