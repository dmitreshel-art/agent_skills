# Структура графа

Хорошая структура графа навыков — это фундамент для масштабируемости и поддерживаемости. Организуйте файлы логически, и граф будет работать годами.

## Базовая структура

```
skill-graphs/
└── [your-graph]/
    ├── INDEX.md                    (точка входа)
    ├── skill-1.md
    ├── skill-2.md
    ├── skill-3.md
    └── ...
```

**Правило:** Одна папка = один граф навыков.

## INDEX.md — точка входа

INDEX.md — это навигационная карта. Должна быть:

### Требования к INDEX.md

1. **Краткое описание:** О чём этот граф (1-2 параграфа)
2. **Организация по темам:** Разделите навыки на группы
3. **Ясные ссылки:** Используйте  для навигации
4. **Quick Reference:** Таблица для быстрого обзора

### Шаблон INDEX.md

```markdown
---
description: "Точка входа в граф [TOPIC]"
version: "1.0.0"
tags: ["tag1", "tag2"]
related: []
---

# Граф навыков [TOPIC]

[Краткое описание — 2-3 строки]

## Основные концепции

-  — Описание
-  — Описание
-  — Описание

## Продвинутые темы

-  — Описание
-  — Описание

## Как навигировать

Начните с  → фундамент. Затем исследуйте по потребностям.
```

### Что включать в INDEX.md

**Обязательно:**
- Название графа
- Краткое описание
- Навигационные ссылки (wiki-links)
- Классификация навыков

**Опционально:**
- Quick reference таблица
- Частые вопросы
- Связи с другими графами

## Организация навыков

### Сгруппирование по уровням

**Уровень 1 (Основные):**
- Фундаментальные концепции
- Начальник должен понимать их
- 3-5 навыков

**Уровень 2 (Продвинутые):**
- Практики и паттерны
- Требуют понимания уровня 1
- 3-5 навыков

**Уровень 3 (Эксперт):**
- Специфические техники
- Специальные случаи
- По необходимости

### Сгруппирование по областям

**Пример для трейдинга:**
```
├── risk-management/    (управление рисками)
├── psychology/          (психология рынка)
├── analysis/           (технический анализ)
└── execution/          (исполнение торгов)
```

**Пример для разработки:**
```
├── fundamentals/        (основные принципы)
├── quality/             (качество кода)
├── collaboration/       (совместная работа)
└── workflow/            (процессы разработки)
```

## Структура файла навыка

Каждый навык — это законченная мысль.

### Секция навыка

```markdown
---
description: "Краткое описание"
version: "1.0.0"
tags: ["tag1", "tag2"]
related: ["skill-1", "skill-2"]
---

# Название навыка

## Overview

[О чём этот навык и почему он важен]

## Core Principles

[3-5 ключевых принципов]

### 1. Принцип

[Объяснение с примерами]

## Common Mistakes

[Чего избегать с плохими/хорошими примерами]

## Quick Reference

[Таблица или чеклист суммирующих ключевые точки]

## Integration

Этот навык связан с:
-  — Объяснение связи
-  — Объяснение связи
```

### Размер файла

- **Минимум:** 500 слов (3-4 секции)
- **Оптимум:** 800-1000 слов (5-6 секций)
- **Максимум:** 1500 слов (8-10 секций)

**Правило:** Лучше меньше, но качественнее.

## Папки и подпапки

### Когда создавать подпапки?

**Создайте подпапки когда:**
- 10+ навыков в графе
- Естественные кластеры навыков
- Разные под-темы требуют организации

**Не создавайте когда:**
- Менее 10 навыков
- Уровни не требуют папок (используйте INDEX.md)

### Пример с подпапками

```
trading-graph/
├── INDEX.md
├── fundamentals/
│   ├── risk-management.md
│   ├── position-sizing.md
│   └── market-psychology.md
├── strategies/
│   ├── momentum-trading.md
│   └── mean-reversion.md
└── tools/
    ├── technical-analysis.md
    └── stop-loss-strategies.md
```

## Кросс-связи между графами

### Связи с другими графами

В INDEX.md добавьте ссылки на другие графы:

```markdown
## Related Graphs

- [[development-graph]] — Граф разработки ПО
- [[trading-graph]] — Граф трейдинга
```

### Общие навыки

Если навык применим к нескольким темам:
- Обратитесь к другим графам
- Избегайте дублирования
- Ссылайтесь на оригинал

## Логика навигации

### Progressive Disclosure

```
INDEX.md (высокий уровень)
    ↓
YAML frontmatter (метаданные)
    ↓
Wiki-links (связи)
    ↓
Полный контент (детали)
```

**Правило:** Агент не должен читать весь файл для простых вопросов.

### MOC (Maps of Content)

Когда навыков > 15, создайте MOC файлы:

```
├── INDEX.md              (главный индекс)
├── psychology-moc.md     (MOC для психологии)
├── strategies-moc.md    (MOC для стратегий)
└── tools-moc.md         (MOC для инструментов)
```

См. [[linking-strategies]] для деталей MOC.

## Лучшая практика

### Начните с малого

**Хорошо:**
```
first-graph/
├── INDEX.md
├── skill-1.md
├── skill-2.md
└── skill-3.md
```

**Плохо:**
```
first-graph/
├── INDEX.md
├── skill-1.md
├── skill-2.md
├── skill-3.md
├── skill-4.md
├── skill-5.md
├── skill-6.md
├── skill-7.md
├── skill-8.md
├── skill-9.md
├── skill-10.md
...
```

**Правило:** Начните с 5-7 навыков, добавляйте позже.

### Не переорганизовывай

После создания, не перестраивайте всё:
- Добавляйте новые навыки в конец
- Создавайте MOC для больших графов
- Не двигайте файлы без причины

**Исключение:** Если текущая структура действительно проблемная.

## Интеграция с файлами системы

### Не трогайте:

- `openspec/` — OpenSpec изменения
- `skills/` — Bundled навыки
- `.openclaw/` — Конфигурация

### Работайте только с:

- `skill-graphs/` — Ваши графы
- Workspace — Ваша рабочая область

## Quick Reference

| Компонент | Размер | Количество |
|-----------|--------|------------|
| INDEX.md | 300-500 слов | 1 на граф |
| Навык | 500-1000 слов | По необходимости |
| Всего в малом графе | 3-5MB | 5-7 навыков |
| Всего в среднем графе | 5-8MB | 8-15 навыков |
| Всего в большом графе | 10-15MB | 15+ навыков |
