# Граф навыков разработки ПО

Software development is more than writing code. It's about writing maintainable, testable, and well-architected systems. Just as developers use patterns and practices to build software, skill graphs give agents structured knowledge to assist with development tasks.

## Core Principles

Foundational knowledge every developer needs:

- [[clean-code]] — Чистый код и принципы написания
- [[testing]] — Стратегии тестирования и качества
- [[debugging]] — Техники отладки и решения проблем

## Collaborative Practices

Working with others effectively:

- [[code-review]] — Практики код-ревью
- [[documentation]] — Стандарты документирования

## Advanced Practices

When foundations are solid:

- [[refactoring]] — Паттерны рефакторинга
- [[architecture]] — Архитектурные принципы
- [[version-control]] — Лучшие практики Git и контроля версий

## Cross-Domain Connections

Development practices connect to other domains:
- [[refactoring]] implements [[clean-code]] principles
- [[testing]] informs [[debugging]] strategies
- [[architecture]] guides [[refactoring]] decisions

## How to Navigate

Start with [[clean-code]] → it's the foundation. Then explore based on your current challenge:
- Need to fix a bug? → [[debugging]]
- Improving code quality? → [[refactoring]] and [[clean-code]]
- Working with team? → [[code-review]] and [[documentation]]
- System design? → [[architecture]]

Each skill references related concepts, so you'll naturally discover interconnected knowledge.
